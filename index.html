DeBERTa

发表于 2022-07-26 分类于 深度学习

简介

DeBERTa是微软在2021年提出的基于BERT的深度学习方法。全称Deconding-enhanced BERT with disentangled attention。主要提出了两个改进方法disentangled attention和enhanced mask decoder(EMD)，以及一个fine-turn的trick。

该方法在GLUE大部分数据集上超过RoBERTa等模型。（提升真的只有一点点…）

自然语言处理（NLP）主要自然语言理解（NLU）和自然语言生成（NLG）。为了让NLU任务发挥最大的作用，来自纽约大学、华盛顿大学等机构创建了一个多任务的自然语言理解基准和分析平台，也就是GLUE（General Language Understanding Evaluation）。

GLUE包含九项NLU任务，语言均为英语。GLUE九项任务涉及到自然语言推断、文本蕴含、情感分析、语义相似等多个任务。像BERT、XLNet、RoBERTa、ERINE、T5等知名模型都会在此基准上进行测试。目前，大家要把预测结果上传到官方的网站上，官方会给出测试的结果。

数据集详细介绍见：GLUE基准数据集介绍及下载

Enhanced Mask Decoder

作者没有同BERT一样在input层加入pos embedding，而是在所有Transformer layers之后，softmax之前，为所有masked了需要预测的token加入绝对位置信息。此时DeBERTa在获得所有相对位置信息，并且仅在解码masked words时加入绝对位置信息，称之为Enhanced Mask Decoder(EMD)。
在比较结果后发现EMD效果要好很多，因此作者推测过早的加入绝对位置信息，可能阻碍了模型对足够相对位置信息的学习。（就不放论文原图了，没看出作者是啥意思…）

fine turn的小技巧

本文提到的小技巧是从虚拟对抗来的。

一般情况下，虚拟对抗学习是一个提高模型泛化能力的正则化方法。其通过使用”对抗实例“来提高一个模型的鲁棒性。而构建”对抗实例“的时候，具体是通过对输入加入small perturbations（轻微扰动）来构造出来的。

在NLP任务中，干扰（perturbation）通常是加在word embedding中，而不是原始文本。所以当模型很大参数很多是，往往方差也会很大，影响对抗训练的稳定性。因此作者提出：applying the perturbations to the normalized word embeddings。即：

  1. SiFT将word emebdding标准化为随机向量（为什么标准化为随机向量呢？）
  2. 在标准化后的word embedding上加入干扰

K-fold交叉验证

交叉验证的定义

交叉验证（Cross Validation），有的时候也称作循环估计（Rotation Estimation），是一种统计学上将数据样本切割成较小子集的实用方法，该理论是由Seymour Geisser提出的。
在给定的建模样本中，拿出大部分样本进行建模型，留小部分样本用刚建立的模型进行预报，并求这小部分样本的预报误差，记录它们的平方加和。这个过程一直进行，直到所有的样本都被预报了一次而且仅被预报一次。把每个样本的预报误差平方加和，称为PRESS(predicted Error Sum of Squares)。

K-fold cross-validation

K折交叉验证，初始采样分割成K个子样本，一个单独的子样本被保留作为验证模型的数据，其他K-1个样本用来训练。交叉验证重复K次，每个子样本验证一次，平均K次的结果或者使用其它结合方式，最终得到一个单一估测。这个方法的优势在于，同时重复运用随机产生的子样本进行训练和验证，每次的结果验证一次，10折交叉验证是最常用的

K-fold交叉验证的目的是什么

如果用一句话来总结：交叉验证是为了评估模型的稳定性，而不是为了从 K 个方程式(model)中选择一个。

具体见下方百度里的解释：

用交叉验证的目的是为了得到可靠稳定的模型。在建立PCR 或PLS 模型时，一个很重要的因素是取多少个主成分的问题。用cross validation 校验每个主成分下的PRESS值，选择PRESS值小的主成分数。或PRESS值不再变小时的主成分数。
常用的精度测试方法主要是交叉验证，例如10折交叉验证(10-fold cross validation)，将数据集分成十份，轮流将其中9份做训练1份做验证，10次的结果的均值作为对算法精度的估计，一般还需要进行多次10折交叉验证求均值，例如：10次10折交叉验证，以求更精确一点。
交叉验证有时也称为交叉比对，如：10折交叉比对

也可以参考此文章，解释的非常透彻： How to choose a predictive model after k-fold cross-validation?

Kaggle使用和打比赛注意事项

使用Kaggle上的Notebook训练

Kaggle上可以免费使用GPU进行模型训练，但输出的数据有保存时效。

可以点击右侧的 Add data > Dataset 上传本地数据，或者 Add data > Competition Data 中获取kaggle上已有的比赛数据

上传数据时，可以一次性upload所有所需数据，输入框 Enter Dataset Title 是为此次上传的所有文件建立的文件夹的名字

需注意：

  1. 文件夹中的“-”，”.“等符号会被忽略，所以在代码中引用文件夹名时需注意

  2. 如果有重复的数据（如vacab.txt会和平台上默认已有的bert_code里的数据重复），完成时会让选择 create without duplicates 新建文件夹不包含重复数据或 include duplicates 包含重复数据。推荐第二个。

  3. 进行一次save version后，dataset里的数据会消失，需要点击add data，选择your dataset中的数据add重新加进来

  4. 注意具体检查数据数目，确保数据无误

上传比赛结果

如果要使用notebook上传比赛结果，则需要满足以下步骤和条件：

  1. 一定保存一版代码： Save Version 并且选择 Save & Run All(Commit)
  2. 保存和运行前，需要先把数据和模型加载到Data中去，具体操作步骤见上文
  3. 运行前需要断开Internet，否则保存后的代码不能被选用上传
  4. 要保证运行的代码，有对测试集的预测并按格式输出文件
  5. 输出的文件，列名和行数都需要分别和 submission.csv 和 test.csv 保持一致
  6. 等程序运行完成后，进入到当前比赛页面下并进入Leaderboard
  7. 点击Submit Predictions后，选择上面步骤中保存的版本 version X 和输出文件 submission.csv

使用vscode远程连接服务器

安装

  1. 在左侧“扩展”里选择并安装扩展 Remote-SSH
  2. 点击左侧新增的扩展按钮，并点击左上角“+”号，然后会出现config的配置，按如下配置
    Host name-as-you-like
    HostName 11.12.205.19
    User root
  3. 此时配置好后尝试登陆，会需要输入密码
  4. 可以用配置成不需要密码
    1. 本地建个ssh ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa-remote-ssh，会生成id_rsa-remote-ssh和id_rsa-remote-ssh.pub两个文件
    2. 将id_rsa-remote-ssh.pub配到服务器的authorized_keys（一般在～/.ssh/）文件里
    3. 将id_rsa-remote-ssh配到本地的config中，即修改刚刚的config成如下
      Host name-as-you-like
      HostName 11.12.205.19
      User root
      IdentityFile ~/.ssh/id_rsa-remote-ssh
  5. 可以连接远程服务器了

断点

使用vscode连接远程服务器，也希望能够进行断点调试。

  1. 进入扩展，搜索python（一般是第一个）

  2. 如果本地还没有安装，可以先点击安装

  3. 如果本地安装了，配备remote-ssh后，会显示“在remote-ssh上安装”，点击按钮进行安装就可以了
